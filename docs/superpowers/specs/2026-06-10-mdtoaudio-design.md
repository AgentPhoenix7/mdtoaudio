# mdtoaudio — Design Spec

**Date:** 2026-06-10
**Status:** Approved

## Overview

A single Python script (`mdtoaudio.py`) that converts an Obsidian markdown note into a natural-sounding audio file using Kokoro TTS, then embeds the audio back into the note as an Obsidian-native voiceover.

The tool is invoked manually — a GUI file picker opens, the user selects a note, and the script handles the rest. No bulk conversion, no watch mode, no automation.

## Usage

```bash
python mdtoaudio.py
```

A tkinter file dialog opens rooted at the vault root. The user selects a `.md` file. The script processes it and exits.

## Internal Functions

```
mdtoaudio.py
├── pick_file()                  → opens tkinter file dialog, returns selected .md path
├── clean_text(md_content)       → produces clean prose for TTS
├── chunk_text(text)             → splits cleaned text into paragraph-based chunks
├── convert_to_audio(chunks, out)→ converts each chunk via Kokoro, concatenates, writes .wav
├── embed_audio(md_path, audio)  → inserts ![[note.wav]] at top of .md if not present
└── main()                       → orchestrates pick → clean → chunk → convert → embed
```

## Text Cleaning Rules

| Element | Example | Action |
|---|---|---|
| Frontmatter | `---\nnext: [[]]` | Strip entirely |
| Embedded files | `![[image.png]]` | Strip entirely |
| Wiki links | `[[My Note]]` | Keep link text → "My Note" |
| Code blocks | ` ```sql ... ``` ` | Keep code text, read as-is |
| Inline code | `` `SELECT *` `` | Keep text, read as-is |
| Dataview blocks | ` ```dataviewjs ... ``` ` | Strip entirely |
| Templater syntax | `<% tp.file.cursor() %>` | Strip entirely |
| HTML tags | `<img src="...">` | Strip entirely |
| Obsidian emoji | `:LiBell:` `:CuLanguage:` | Convert to humanized name → "Bell", "Language" |
| Bold / italic | `**text**`, `*text*` | Keep text, strip markers |
| Headers | `## What you'll learn` | Keep text, strip `#` symbols |
| Tables | `\|col\|col\|` | Strip entirely |
| Horizontal rules | `---` | Strip |

**Obsidian emoji handling:** Strip the prefix (`Li`, `Cu`, `Lu`, etc.) and the surrounding colons, then insert a space before each capital letter to produce a readable word (e.g., `:LiBell:` → "Bell", `:CuLanguage:` → "Language", `:LiCheckCircle:` → "Check Circle").

## Audio Output

- **Format:** `.wav` (Obsidian supports WAV natively; avoids ffmpeg dependency)
- **Location:** Same directory as the source `.md` file
- **Filename:** Same stem as the note (e.g., `What is Programming.md` → `What is Programming.wav`)
- **TTS engine:** Kokoro (local, free, no API key)

## Audio Embedding

After generating the audio, the script inserts an Obsidian embed at the very top of the `.md` file:

```markdown
![[What is Programming.wav]]

# What Is Programming...
```

- If the embed already exists → skip (idempotent)
- If an embed for a different audio file exists at the top → replace it
- The rest of the note content is untouched

## Dependencies

| Package | Purpose |
|---|---|
| `kokoro` | Local natural-sounding TTS |
| `soundfile` | Write generated audio to `.wav` |
| `tkinter` | GUI file picker (built into Python stdlib) |

Project is set up with `uv`:
```bash
uv init mdtoaudio
uv add kokoro soundfile
uv run mdtoaudio.py
```

A `pyproject.toml` is generated by `uv init` and dependencies are tracked there. No manual `requirements.txt` needed.

## Chunking

Long notes are split into chunks at paragraph boundaries before TTS conversion. Each chunk is converted independently and the resulting audio arrays are concatenated before writing the final `.wav`. This prevents Kokoro from failing or degrading on very long inputs.

## Progress Feedback

The script prints simple status lines to stdout during processing:

```
Cleaning text...
Generating audio (chunk 1/5)...
Generating audio (chunk 2/5)...
...
Embedding audio in note...
Done → /path/to/What is Programming.wav
```

## Error Handling

- If the user cancels the file picker → exit silently
- If the selected file is not `.md` → print an error and exit
- If TTS fails → print error, do not modify the `.md` file
- If the `.md` file is not writable → print error and exit

## Non-Goals

- No watch mode
- No bulk/folder conversion
- No Docker
- No cloud TTS APIs
- No GUI beyond the file picker
